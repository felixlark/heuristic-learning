# 一小时理解直觉学习

这是一条适合自学、技术分享和读书会的快速路线。它回答四个问题：什么是 Heuristic Learning，它和深度学习、强化学习有什么不同，代码规则怎样从反馈中进化，以及这种方法目前有哪些证据边界。

## 1. 从一个熟悉的问题开始

许多智能系统并不只有模型。模型外还有阈值、工具选择、状态机、重试策略、测试、记忆和工作流。系统失败时，工程师通常修改这些软件结构。编码智能体出现后，这种修改也可以进入持续反馈循环。

HL 研究的正是这个循环：**让智能体从环境结果中发现失败，再修改可读的软件结构，并用测试确认旧能力没有被破坏。**

## 2. 三种学习更新什么

| 方法 | 主要更新对象 | 常见反馈 | 适合的问题 |
| --- | --- | --- | --- |
| 深度学习 | 神经网络权重 | 数据集损失 | 感知、表示和生成 |
| 强化学习 | 策略或价值函数参数 | 环境奖励 | 连续决策与长期回报 |
| Heuristic Learning | 规则、状态机、工具调用、测试和记忆 | 日志、回放、测试、奖励和人工评价 | 可解释的软件系统维护 |

它们不是互斥关系。HL 可以调用模型，也可以把强化学习策略蒸馏成可读结构；判断重点是“这一轮究竟更新了什么”。

## 3. 看见一次最小更新

从 [GridWorld 示例](/zh-cn/examples/) 开始：基线策略只朝目标贪心移动，因此会进入局部陷阱。改进策略增加陷阱检测和安全动作，随后重新运行同一批测试。这里学到的不是某条规则，而是一种可重复的方法：

1. 保存失败场景。
2. 写出最小、可解释的修改。
3. 比较修改前后的结果。
4. 把失败和成功都写入记录。
5. 用旧场景做回归检查。

## 4. 当前证据说明什么

Jiayi Weng 的公开 benchmark 已把这一思路扩展到 CartPole、MountainCar、Acrobot、LunarLander 和 BipedalWalker。结果显示透明策略能够解决其中多项任务，也暴露了结构修改会退化、短种子搜索会过拟合、复杂任务可能需要教师蒸馏等问题。

这些结果支持“软件结构可以成为学习对象”这一研究方向，但还不足以证明 HL 能普遍替代 RL 或 DL。详细方法和限制见 [真实环境 Benchmark](/zh-cn/benchmark/)。

## 5. 接下来动手

- 用 15 分钟跑通 [最小示例](/zh-cn/examples/)。
- 用 20 分钟读一个 [案例](/zh-cn/cases/)。
- 用 20 分钟修改一条策略并运行测试。
- 用最后 5 分钟写下失败、修改、结果和下一步假设。
