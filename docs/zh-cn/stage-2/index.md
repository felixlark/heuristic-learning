# HL 基础概念

## Heuristic Learning

Heuristic Learning 是一个学习过程。它的关键不是“人手写规则”，而是规则、策略、测试、记忆和实验记录可以被编码智能体持续维护。

HL 的最小元素：

| 元素 | 含义 |
| --- | --- |
| 状态 | 环境观测、日志、测试输出、回放、人工反馈 |
| 动作 | 程序策略做出的选择 |
| 反馈 | 奖励、错误、失败轨迹、测试结果、评审意见 |
| 更新对象 | 策略代码、状态检测器、配置、测试、记忆、文档 |
| 更新者 | 能读取上下文并修改代码的智能体 |

## Heuristic System

Heuristic System 是 HL 长期维护出来的对象。一个孤立的 `policy.py` 还不是 HS；它至少需要连接：

- 程序化策略
- 状态表示
- 反馈通道
- 实验记录
- 测试或回放
- 更新机制
- 维护记忆

## 与手写规则的区别

传统规则常见问题是维护成本高、冲突难查、迭代慢。HL 的假设是：当编码智能体足够强，维护曲线会改变。原本“不值得长期维护”的启发式规则，可能变成可持续改进的软件资产。

## HL 的遗忘

HL 也会遗忘，只是遗忘形式更像工程事故：

- 新规则修复一个场景，却破坏旧场景。
- 记忆写错后长期误导智能体。
- 测试太窄，策略学会钻测试空子。
- 规则堆叠太多，系统变得不可维护。

因此 HL 课程必须强调测试、回放、实验记录和回归验证。
